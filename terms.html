Termos de hipertexto

Este é um glossário de termos usados dentro do projeto WWW. Na maioria dos casos, seu uso corresponde ao uso converncional em círculos de hipertexto. Âncora Uma área dentro de um conteúdo de um nó que é a fonte ou destino de um link . O âncora pode ser o conteúdo do nó inteiro. Normalmente, clicar em um mouse em uma área de âncora faz com que o link seja seguido, deixando a âncora na extremidade oposta do link exibido. Âncoras tendem a ser destacadas de forma especial (sempre, ou quando o mouse está sobre elas), ou representadas por um símbolo especial. Uma âncora pode, e muitas vezes corresponder a todo o nó. (também às vezes conhecido como "span", "região", "botão" ou "extensão"). Anotação A vinculação de um novo nó de comentário a um nó existente. Se os leitores podem anotar nós, então eles podem fornecer feedback imediatamente se as informações forem enganosas, desatualizadas ou simplesmente erradas. Assim, a qualidade das informações na web pode ser melhorada. (Mais...) Criação Um termo para o processo de redação de um documento. A "autoria" parece ter sido usada para enfatizar que a produção de documentos envolvia mais do que apenas escrever. Link de volta Um elo em uma direção implicava a partir da existência de uma liminar explícita na outra direção. Veja: Construindo back-links Navegador Um programa que permite que uma pessoa leia hipertexto . O navegador dá alguns meios de visualizar o conteúdo de nódulos , e de navegar de um nó para outro. Botão Uma âncora que é a fonte de um link. Muitas vezes, mas nem sempre, representado na tela para parecer um botão. Cartão Um termo alternativo para um nó em um sistema (por exemplo, HyperCard, Notecards) no qual o tamanho do nó é limitado a uma única página de um tamanho limitado. Cliente Um programa que solicita serviços de outro programa. Normalmente, o navegador é cliente de um servidor de dados. Base de dados Usamos isso vagamente como termo para uma coleção de nós. Imaginamos informações de gestão para um desses sendo mantidos em um só lugar e todos sendo acessíveis pelo mesmo servidor. Os links fora disso são "externos", e os que estão dentro são "internos". Não implicamos nada sobre como as informações devem ser armazenadas. Dimons Um programa que é executado independentemente, por exemplo, do navegador . Os Daemons podem executar várias tarefas de gerenciamento, como índices de construção, visões gerais e back-links. Sob unix, "daemon" é usado para " servidor ", porque os servidores normalmente são executados independentemente. Documento Um termo para um nó em alguns sistemas (por exemplo, Intermedia). Às vezes usado por outros como termo para uma coleção de nódulos sobre temas relacionados, possível armazenado ou distribuído como um. O termo preferido na documentação W3. Domínio Usamos isso especificamente para uma unidade de proteção. Poderia possivelmente corresponder a um banco de dados , e nesse caso seria um termo melhor (menos vago) para ele. Externo Um link para um nó em um banco de dados diferente. Ver banco de dados Anfitrião Um computador em uma rede. Usamos este termo em vez do termo " nó " que é frequentemente usado para um documento em uma web de hipertexto . Hipermídia Hypertext MultiMedia . HyperMedia e HyperText tendem a ser usados livremente no lugar um do outro. Outras mídias que não sejam textos geralmente incluem gráficos, som e vídeo. (Mais...) Hipertexto Texto que não é constrangido a ser linear. (Mais...) Índice Algo que aponta para outros dados; uma instalação de servidor que fornece indicadores para determinados dados em função de uma consulta; uma tabela de conteúdo de um livro em forma de hipertexto. (Mais). Interno Um link para um nó no mesmo banco de dados. Veja o banco de dados . Link Uma relação entre duas âncoras , armazenada no mesmo ou diferente banco de dados. Consulte "Interno" e "Externo". Navegação O processo de mudança de um nó para outro através da web de hipertexto. Isso normalmente é feito seguindo links . Vários recursos de um navegador em particular podem tornar isso mais fácil. Isso inclui manter um histórico de onde o usuário esteve e desenhar diagramas de links entre nós próximos. (Mais...) Nodo Uma unidade de informação. Também conhecido como quadro (KMS), cartão (Hypercard, Notecards). Usado com este significado especial em círculos de hipertexto: não confunda com "nó" que significa "host de rede". Para os benefícios do usuário, usamos o termo " documento ", pois este é o termo mais próximo fora do mundo do hipertexto. Proteção A prevenção de usuários não autorizados de ler, ou escrever, um determinado pedaço de dados. Também conhecido como "autenticação", "controle de acesso", etc. (Mais...) Caminho Um conjunto ordenado de nós ou âncoras que representam uma sequência na qual uma web pode ser lida. Um caminho pode representar a sequência que um leitor realmente usou, ou pode ser uma sequência recomprida ao leitor pelo autor. Leitor Usamos este termo para a pessoa que navega, para distingui-lo do programa (navegador) que ele usa. Servidor Um programa que presta um serviço a outro, conhecido como cliente. Em um sistema de hipertexto, um servidor fornecerá informações de hipertexto a um navegador . Veja também: daemon . Rastreamento A localização automática de nódulos por navegação automática . Exemplos podem ser encontrar todos os nós dependentes de outro nó, todas as pessoas interessadas em um determinado nó, todos os módulos que usam um determinado módulo. Outro exemplo é um traço que começa com mais de um nó, como encontrar um nó em comum entre dois grupos, ou caminho ligando dois nós. Topologia A conectividade permitida entre nós, âncoras e links: por exemplo, mapeamentos 1-1 ou muitos 1. (Mais...) Versionação O armazenamento e o gerenciamento de versões anteriores de uma informação, para segurança, diagnóstico e interesse. Isso é importante quando muitos usuários podem editar o mesmo material. (Mais...) Teia Um conjunto de nódulos interligados por links . Muitas vezes, o conjunto de todos os nós que estão interligados. Veja também Topologia .
