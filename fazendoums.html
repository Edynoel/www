W3 Server Software

Um servidor W3, como o ftp daemon, é um programa que responde a uma conexão tcp de entrada e fornece um serviço ao chamador. Existem muitas variedades de software de servidor W3 para servir diferentes formas de dados.

O daemon básico

O programa básico de daemon W3 serve arquivos já em hipertexto ou texto simples. Este daemon então é usado como base para muitos outros tipos de servidor e gateway. A documentação inclui:
  • Guia do usuário .
  • Internas, uma descrição do código.


    • Grande parte do acima também pode se aplicar no todo ou em parte a outros servidores mencionados abaixo. Seja qual for o servidor que você está executando, você provavelmente estará interessado em:
  • Ferramentas para provedores de informações

    • Fazendo um novo servidor

    Este daemon é frequentemente usado como base para um servidor mais específico para um determinado aplicativo. Um servidor que permite que um mundo de dados seja visto como parte do universo W3 é conhecido como um gateway. (A maioria dos servidores poderia, portanto, ser considerada como gateways, mas o termo implica alguma conversão ou mapeamento entre mundos diferentes) . Para tutoriais curtos com exemplos, consulte:

    Outros servidores e Gateways

    São servidores que fornecem dados extraídos de outros sistemas. eles são construídos usando código do daemon básico, ou scripts. Find gateway para CERN/VM XFIND, que chama um executivo REXX para obter as informações do sistema XFIND em execução no mainframe CERNVM. Gateway de ajuda do VMS Isso permite que quaisquer arquivos de ajuda VMS sejam disponibilizados aos clientes www. É executado no VAX/VMS. WAISGate Uma porta de entrada para informações disponíveis usando o protocolo W.A.I.S. DCLServer Um servidor para sistemas VMS que permite que você escreva um gateway para o seu próprio sistema de informações favorito usando DCL.
